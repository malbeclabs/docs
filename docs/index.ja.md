# DoubleZeroへようこそ

DoubleZeroは、ブロックチェーンなどの分散システム向けに最適化された高性能分散型プロトコルおよびネットワークです。DoubleZeroのユーザーは、分散システムデータの高度に最適化された伝送と前処理のための統合サービスを利用できます。

## DoubleZeroは誰のためのものですか？

### コントリビューター

DoubleZeroのコントリビューターは、十分に活用されていないプライベートファイバーリンクやハードウェア（スイッチやFPGAデバイスなど）の形でネットワーク容量を保有しています。これらのリソースをDoubleZeroネットワークの拡張に活用するよう構成することで、統合された情報サービスが構築されます。DZプロトコルを使用して、ネットワークコントリビューターはこのネットワーク容量を動的かつ広範なグローバルメッシュネットワークに組み上げ、その貢献に対してトークンインセンティブを受け取ります。

### ユーザー

DoubleZeroのユーザーは、高性能なネットワーク接続を必要とするブロックチェーンなどの分散システムに参加しています。ブロックチェーンシステムをDoubleZeroと相互接続することで、ユーザーはネットワークI/Oを加速するための最適化機能のバンドルを利用できます。

初期実装において、DoubleZeroネットワークはブロックチェーンのバリデータークライアント、およびRPCやMEVインフラストラクチャプロバイダーと互換性があります。これらのユーザーシステムは、ネットワークへのアクセスを許可する前に、それぞれのレジャー上で一意に識別する公開鍵アドレスを検証することでアドミッションコントロールを受けます。ネットワークが他のエコシステムのユーザーシステムを含むように進化するにつれて、各ユーザープロジェクトコミュニティに合わせた追加の識別および認証戦略が追加される予定です。

## メリット

DoubleZeroの唯一の代替手段はパブリックインターネットです。パブリックインターネットと比較して、DoubleZeroネットワークはブロックチェーンに対して2つの改善を提供します。

*第一に*、受信トランザクションは、DoubleZeroネットワークのユーザー（ブロックチェーンノードなど）に送信される前に、ネットワークコントリビューターのハードウェアレベルで非裁量的な方法でエッジフィルタリングされます（スパムや重複の除去など）。これにより、各バリデーターが十分なリソースをプロビジョニングする必要がなく、ブロックチェーンはシステム全体で共有されたフィルタリングリソースの恩恵を受けることができます。その結果、バリデーターは直接収益を生み出す活動にリソースを再配分できます。この機能はまだ実装されていません。

*第二に*、送信メッセージはより直接的にルーティングされ、効率を向上させるために優先されます。これは、DoubleZeroではグローバルデータ移動のルーティングロジックがパブリックインターネットとは異なり速度に最適化されているために可能です。したがって、バリデーターはジッター（2つのデータケーブルエンドポイント間のレイテンシーのばらつきの量として定義される）の削減の恩恵を受けます。本質的に、ブロックチェーンノードはブロックの送受信をより高速に行うことができます。

これらのDoubleZeroネットワークのメリットを組み合わせることで、パブリックインターネットでは達成不可能なレベルのパフォーマンスを実現できます。

詳細については、[DoubleZero Protocol](https://doublezero.xyz/whitepaper.pdf)ホワイトペーパーをご覧ください。
