# サイト廃止ガイド（コントリビューター向け）

本ガイドでは、DoubleZero Device（DZD）の廃止またはサイトからの撤退方法について説明します。接続中のユーザーに影響を与えることなく、デバイスとリンクをネットワークから削除し、オンチェーンで削除する手順を解説します。

プロセスは3つの段階で進行します。廃止日の31日前にデバイスの上限を設定し、通知期間中に接続ユーザーに移行を通知して移動の猶予を与え、廃止日にリンク、インターフェース、デバイスのドレインと削除を実施します。

> ⚠️ **まずDoubleZeroと調整してください：**
> デバイスやサイトを廃止する前に、必ずDoubleZeroチームと事前に調整してください。デバイスの上限設定やリンクのドレインを行う前に、日程と計画について合意し、ユーザー移行およびファウンデーション側で必要な手順を調整できるようにしてください。

---

## 概要

| タイミング | アクション | 担当者 |
|------|--------|-----|
| 31日前 | 新規ユーザーが接続できないようデバイスの上限を設定（`--max-users 0`） | コントリビューター |
| 14日前 | 接続中のユーザーに別のデバイスへの移行を通知 | DoubleZeroチーム |
| 通知期間 | ユーザーが自ら他のDZDに再接続 | ユーザー |
| 廃止日 | リンク、インターフェース、デバイスのドレインと削除 | コントリビューター |

原則：

- **新規ユーザーを早期に停止し、既存ユーザーは段階的に移行。** デバイスの上限を早期に設定することで、その時点以降はユーザーが減少するのみとなります。既存ユーザーは引き続き利用可能で、各自のスケジュールで移行します。
- **通知期間中はすべてを稼働状態に保つ。** 廃止日までリンクやデバイスのドレインは行わないでください。移行中のユーザーが通常通りサービスを利用できるようにするためです。
- **解体の順序はコントラクトにより強制されます。** アクティブ状態のリンクやデバイスは削除できないため、以下の手順ではまずドレインを行い、最後に削除します。

> ⚠️ **緊急の場合：**
> 撤退まで31日未満の場合は、直ちに開始してください。今すぐデバイスの上限を設定し、利用可能な期間に合わせてウィンドウを短縮してください。手順の順序は変わりません。

---

## フェーズ1 — デバイスの上限設定（31日前）

新規ユーザーが接続できないようデバイスの上限を設定します：

```bash
doublezero device update --pubkey <DEVICE_PUBKEY> --max-users 0
```

既存ユーザーには影響がなく、引き続き利用できます。サイトで廃止対象となるすべてのデバイスに対して繰り返し実行してください。リンクとデバイスは完全にアクティブな状態を維持するため、接続中のユーザーは通常通りサービスを利用できます。

---

## フェーズ2 — 通知期間（14日前）

DoubleZeroチームが接続中のユーザーに通知し、廃止日までに別のDZDに再接続するよう依頼します。どのユーザーに誰が連絡するかについて、チームと調整してください。

この期間中はドレインを行わないため、ユーザーは通常通りサービスを利用できます。ユーザーは各自のペースで再接続します。以下のコマンドでユーザー数を監視してください：

```bash
doublezero device list
```

---

## フェーズ3 — 廃止日

以下の手順を順番に実行してください。各手順が次の手順を解除します。

> ⚠️ **最終的なデバイス削除の前に：**
> 最後の手順を実行する前にDoubleZero Foundationに通知してください。Foundationは期限内に移行しなかったユーザーを削除します（これを行わないと削除がブロックされます）。また、ファウンデーション側で必要な手順も完了します。

### 1. リンクのドレイン

まずソフトドレイン、次にハードドレインを実行します。各状態の詳細については[Link Draining](contribute-operations.md#link-draining)を参照してください。

```bash
doublezero link update --pubkey <LINK_PUBKEY> --status soft-drained
# トラフィックが移動したら：
doublezero link update --pubkey <LINK_PUBKEY> --status hard-drained
```

削除対象のデバイス上のすべてのリンクに対して繰り返し実行してください。

### 2. リンクの削除

```bash
doublezero link delete --pubkey <LINK_PUBKEY>
```

これにより、リンクが使用していたインターフェースが解放されます。

### 3. インターフェースの削除

```bash
doublezero device interface delete <DEVICE_CODE> <INTERFACE_NAME>
```

デバイス上の各インターフェースに対して繰り返し実行してください。

### 4. デバイスのドレイン

```bash
doublezero device update --pubkey <DEVICE_PUBKEY> --status drained
```

ドレインにより、デバイスがルーティングから除外され、残存するユーザーセッションがすべて終了します。また、デバイスをアクティブ状態から移行させ、削除可能な状態にします。

### 5. デバイスの削除

```bash
doublezero device delete --pubkey <DEVICE_PUBKEY>
```

デバイスの削除は、アクティブ状態でなくなり、参照しているリンクがなく、インターフェースが残っていない場合にのみ可能です。これらは前の手順で処理されています。

---

## キャンセルまたは延期

上限設定とドレインは、削除を開始するまで元に戻すことができます：

```bash
doublezero device update --pubkey <DEVICE_PUBKEY> --max-users <ORIGINAL_VALUE>   # 容量を復元
doublezero link update --pubkey <LINK_PUBKEY> --status soft-drained             # ハードドレインから戻す
doublezero link update --pubkey <LINK_PUBKEY> --status activated                # アクティブに戻す
doublezero device update --pubkey <DEVICE_PUBKEY> --status activated            # デバイスのドレインを解除
```

リンク、インターフェース、またはデバイスの削除は永続的であり、オンチェーンアカウントが閉鎖されます。撤退が確定してから削除を開始してください。
